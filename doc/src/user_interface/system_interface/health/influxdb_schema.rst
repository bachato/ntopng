InfluxDB Schema
--------------------

An InfluxDB schema refers to the structure of a time-series database.
Below is a list of some examples of the time-series schemas contained in ntopng. 
Each table below contains four columns:
- Schema: Represents the unique identifier of the schema
- Tags: Lists relevant identifiers or attributes associated with the schema
- Step: Indicates the time interval (in seconds) at which the metric is measured
- Metrics: Details the specific data points or measurements tracked, such as counts, traffic volume, or packet details.
- Description: Describes the schema
In ntopng, there are five classifications based on the length of the step for the time-series data.

List of schemas with a step of 1 second: scripts/lua/modules/timeseries/schemas/ts_second.lua

| Schema                                     | Tags                      | Step           | Metrics                               | Description                                                                     |
+============================================+===========================+================+=======================================+=================================================================================+
| iface:traffic                              | ifid                      | 1              | bytes                                 | Total traffic in bytes per interface                                            |
+--------------------------------------------+---------------------------+----------------+---------------------------------------+---------------------------------------------------------------------------------+
| iface:traffic_rxtx                         | ifid                      | 1              | bytes_sent, bytes_rcvd                | Bytes sent and received per interface                                           |
+--------------------------------------------+---------------------------+----------------+---------------------------------------+---------------------------------------------------------------------------------+
| iface:traffic_ip                           | ifid                      | 1              | bytes_ipv4, bytes_ipv6                | IPv4 and IPv6 traffic per interface                                             |
+--------------------------------------------+---------------------------+----------------+---------------------------------------+---------------------------------------------------------------------------------+
